# 2026-02-13 - ESP32 Data Logger Hardware Design

## PCB Review Completed
- Reviewed ESP32_Board_REV3 KiCad files
- Analyzed POWER, Interface connector, and main schematics
- 4-layer board (F.Cu, In1.Cu, In2.Cu, B.Cu) - appropriate for ESP32 WiFi

## BOM Optimization Decisions

### Components to Remove (~$9 savings)
- **ICS-43434** - MEMS microphone (not needed for airflow/temp logging)
- **BQ27441** - Battery fuel gauge (optional for this use case)
- **Status LEDs** (CHG1, PGOOD1, PG, LD1) - save power and cost
- **R29 NTC** - Thermistor related to fuel gauge

### Components to Add (~$5)
- **DS3231MZ+ RTC** - SOIC-16 package, I2C interface, ±5ppm accuracy
- **32.768kHz crystal** - FC-135 package for RTC
- **CR2032 SMD battery holder** - Keystone or similar, 20mm diameter
- Supporting passives (decoupling caps, pull-ups)

### I2C Address Verification
All devices have unique addresses - no conflicts:
- Airflow sensor: 0x40
- SHT30 (Temp/RH): 0x44
- DS3231 RTC: 0x68

## Physical Design
- DS3231 placement: Bottom-left on B.Cu near battery connector (~82mm, -115mm)
- CR2032 holder: Adjacent on B.Cu (~85mm, -122mm)
- Height addition: ~4mm total - need to verify enclosure clearance
- Crystal: Place close to DS3231 (<10mm trace length)

## Key Findings
- Net BOM cost reduction: ~$4 savings
- Get reliable timestamp across power cycles
- 4-layer board appropriate - keep for WiFi/BLE performance
- Power traces need widening: 0.5mm for 3.3V, 0.8mm for 5V
