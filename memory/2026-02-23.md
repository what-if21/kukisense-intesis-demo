# 2026-02-23 - E-ink Frame Firmware Updates

## Summary
Multiple firmware improvements and bug fixes for the ESP32-S3 GDEP073E01 7-color e-ink IAQ display project.

## Changes Made

### 1. Configuration Screen Simplified
**Problem:** Config screen showed too much technical info and outdated GPIO reset instructions.

**Solution:** Simplified to clean, user-friendly display:
- Header: "E-INK FRAME" (blue bar)
- Main message: "Not Configured"
- Instructions: "Configure using mobile app"
- Removed: All parameter listings, GPIO reset instructions

**File:** `main/main.c` - `draw_config_screen()` function

### 2. Display Parameters Fix
**Problem:** `display_params` field was empty in NVS config, causing no tiles to show on IAQ dashboard.

**Root Cause:** Config blob may have been saved with older firmware version or field wasn't initialized properly.

**Solution:** Added fallback in `load_config()`:
```c
if (strlen(g_config.display_params) == 0) {
    ESP_LOGW(TAG, "Display params empty, setting defaults");
    strcpy(g_config.display_params, "co2,pm25,pm10,tvoc,temperature,humidity");
}
```

**Result:** All 6 tiles now display correctly (CO2, PM2.5, PM10, TVOC, Temperature, Humidity).

### 3. Force Display Update After WiFi
**Problem:** Display showed placeholder data after boot, real data only appeared after 5-minute auto-refresh cycle.

**Solution:** Added forced display refresh immediately after fetching data from ThingsBoard:
```c
if (server_fetch_data()) {
    // ... update timestamp ...
    // Force display update with new data
    draw_iaq_with_text();
}
```

**Result:** Real sensor data appears within seconds of WiFi connection.

### 4. GPIO 46 Issue Resolved (Earlier)
**Problem:** GPIO 46 has fixed internal pulldown on ESP32-S3, making it unusable for button detection.

**Solution:** Changed config reset button from GPIO 46 → GPIO 41.

**Button Mapping:**
| Function | GPIO |
|----------|------|
| Refresh | 9 |
| Sleep/Wake | 42 |
| Config Reset | 41 |

### 5. Config Reset Button (Runtime)
**Feature:** Config reset (GPIO 41) now works at any time, not just during boot.

**Behavior:**
- Hold GPIO 41 for 3 seconds → Clears NVS config
- Shows "CONFIGURATION CLEARED!" confirmation
- Auto-reboots to BLE setup mode
