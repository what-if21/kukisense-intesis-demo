## ESP32-S3 2.8" Touch Display - Code Review & Updates

### Repository Created
- **Repo**: https://github.com/what-if21/esp32s3_2.8touch
- **Initial commit**: `55dab58` - ESP32-S3 display with LVGL, RGB LCD, touch, IMU

### Hardware
- ESP32-S3 (480x480 RGB LCD, ST7701S driver)
- GT911 capacitive touch
- QMI8658 IMU (accel + gyro)
- PCF85063 RTC, SD card, buzzer

### Code Review Findings
- Good architecture: System_Task + GUI_Task with semaphore
- Clean driver separation
- Missing: error handling, watchdog feeds, hardcoded UART config
- No power management

### Improvements Implemented (Commit `662f1a4`)

**1. IMU Error Handling (`QMI8658.c/h`)**
- All functions return `esp_err_t`
- Device detection with 3 retries
- WHO_AM_I validation (expect 0x05)
- I2C error logging with ESP_LOG
- Timeout handling for CTRL9 commands

**2. UART Kconfig Options**
- `CONFIG_SENSOR_UART_NUM` - UART port (0-2)
- `CONFIG_SENSOR_UART_BAUD_RATE` - baud rate
- `CONFIG_SENSOR_UART_TX/RX_PIN` - GPIO pins
- `CONFIG_SENSOR_UART_BUF_SIZE` - buffer size

**3. Power Management**
- Auto-backlight dimming (Kconfig)
- `CONFIG_ENABLE_AUTO_BACKLIGHT` - enable/disable
- `CONFIG_BACKLIGHT_DIM_TIMEOUT` - seconds of inactivity
- `CONFIG_BACKLIGHT_DIM_LEVEL` - dimmed brightness %
- Touch activity wakes display
- Timer checks every 10 seconds

**4. Watchdog Support**
- `esp_task_wdt_reset()` in QMI8658_Init and QMI8658_Loop
- Prevents WDT timeout during sensor operations

### Files Modified
- `main/QMI8658/QMI8658.c` - Error handling, watchdog
- `main/QMI8658/QMI8658.h` - Updated function signatures
- `main/LCD_Driver/ST7701S.c/h` - Auto-dimming functions
- `main/Touch_Driver/GT911.c` - Activity notification
- `main/LVGL_NEWUI/ui.c` - Use Kconfig UART settings
- `main/Kconfig.projbuild` - New configuration options

### UI Code Review & Fixes (Commit `b5f4b52`)

**Review Findings:**
- cJSON memory leak potential
- No bounds checking on sensor values
- Instant value changes (no animations)

**Fixes Implemented:**

**1. cJSON Memory Management**
- Add proper `cJSON_Delete(root)` after parsing
- Add JSON structure validation (check for `{` and `}`)
- Log warnings for invalid JSON frames
- Validate JSON root is an object before processing

**2. Sensor Value Bounds Checking**
- Add `sensor_min[]` and `sensor_max[]` arrays:
  - PM2.5: 0-999, Temp: -40 to 80°C
  - RH: 0-100%, CH2O: 0-10 mg/m³
  - CO2: 0-5000 ppm, TVOC: 0-5000 ppb
  - PM10: 0-999
- Add `clamp_sensor_value()` helper function
- Validate all values before display
- Support multiple JSON key variants (PM25/pm2.5/PM2.5, Temp/temperature/Temperature)
- Keep previous value if new value is out of range

**3. Smooth Animations**
- Add `lv_anim_t` structure for arc animation
- `animate_arc_to()` function for gauge transitions
- 300ms ease-out animation path
- Proper arc range setting (0-100) with bounds checking
