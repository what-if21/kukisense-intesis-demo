# 2026-02-20 - E-ink Frame Development

## Timezone Bug Fix
**Problem:** Time display showed "-307:03 UTC +420" - completely wrong hour value and timezone format.

**Root Cause:** 
- POSIX TZ format uses inverted signs (UTC+7 means UTC minus 7 hours)
- Simple longitude/15 calculation gave Singapore UTC+7 instead of UTC+8

**Solution:**
- Created `timezone_table[]` with 25 major Asian cities/countries
- Added `find_nearest_timezone()` function using distance calculation
- Singapore (1.32°N, 103.90°E) now correctly matches Singapore entry (1.35°N, 103.82°E)
- Displays "Updated: 16:30 SGT" with correct UTC+8 offset

**Timezone table includes:**
- Southeast Asia: Singapore, KL, Jakarta, Bangkok, Manila, Ho Chi Minh, Hanoi, Phnom Penh, Vientiane, Yangon, Brunei
- East Asia: Hong Kong, Taipei, Tokyo, Seoul, Beijing, Shanghai, Guangzhou, Shenzhen, Macau, Osaka
- South Asia: Mumbai, Delhi, Karachi, Dhaka, Colombo, Kathmandu
- Australia/Oceania: Sydney, Melbourne, Perth, Brisbane
- Middle East: Dubai, Riyadh, Tehran, Tel Aviv
